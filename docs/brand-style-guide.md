# The Sweet Fork Brand Style Guide

## Official Name

- Use `The Sweet Fork` in public headings, body copy, metadata, alt text, structured data, and social preview text.
- Avoid standalone `Sweet Fork` in customer-facing copy unless space is severely constrained or it is part of an internal/admin label.
- Use `The Sweet Fork's` for possessive references.

## Voice

- Premium, calm, editorial, and specific.
- Lead with tailored inquiry, limited availability, polished finish, and thoughtful hospitality.
- Avoid generic bakery filler, demo language, or anything that makes the public site feel like a prototype.

## CTA Language

- Preferred primary action: `Start Your Inquiry`.
- Support CTAs with low-friction reassurance such as `No commitment required` or reply timing when accurate.
- Do not make the inquiry flow sound like a technical quote engine or ecommerce checkout.

## Alt Text

- Describe the dessert, finish, and context in one concise sentence.
- Use specific materials, colors, and event context when visible.
- Do not start with `image of` or repeat adjacent captions.
- Decorative flourishes and purely redundant marks should use empty alt text when rendered as images.

## Metadata

- Route titles and social titles should include or inherit `The Sweet Fork`.
- Descriptions should mention the route-specific offering, Centerville or Northern Utah context where useful, and the inquiry-first model.
- Avoid duplicate titles across public routes.

## Visual System

- Preserve the ivory, cream, charcoal, gold, rose, and stone palette.
- Keep typography elegant and readable; use large serif display type only where the layout has room.
- On mobile, prioritize fast access to product categories, gallery examples, and inquiry CTAs.
- Do not introduce generic stock-style visuals, harsh colors, or overly technical dashboard patterns on public pages.
